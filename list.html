Product Requirements — School Finance & Operations

This document outlines the pages and components to be developed for the International School finance and operations system. Each card links to the expected page or module.

1. Core Financial

The end of this flow will be Financial Summary. Accountant can record Income and Expense with ease, and reports will be created by system.
Financial Summary: Automatic
Add New Student (Accountant): Accountant can create new student, only core info + financial matter
Student Statement: Per Student
Student Summary: ALL Students

2. Academic

The end of this flow will be Student Report Card. Once data input of all three sections done for all subjects, Principal can click the "Generate Student Progress Report" -> only then the report can be seen.
Data input per subject:
Section 1: Quantitative
Section 2: Qualitative (Proficiency)
Section 3: Comment
Generate Student Progress Card: list of all subject scores data entry status
Data Entry for Section 1: Quantitative
Data Entry for Section 2: Qualitative (Proficiency)
Data Entry for Section 3: Comment
Student Statement: Per Student
Student Summary: ALL Students

3. Payroll

The end of this flow will be salary summary per month (payroll is calculated automatically)

This page is a functional requirements index. Styling is indicative; developer is free to implement with React, Vue, server-rendered HTML, or static pages as appropriate.
