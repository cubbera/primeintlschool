Product Requirements — School Finance & Operations

This document outlines the pages and components to be developed for the International School finance and operations system. Each card links to the expected page or module.

1. Core Financial

The end of this flow will be Financial Summary. Accountant can record Income, Expense, and other transactions with ease, and reports will be created by system.

• Student Enrollment & Tuition Plan (Accountant): Accountant can create new student, only core info + financial matter
• Create Invoice & Payment (Accountant): Accountant can create invoice and payment
• Student Statement: Per Student
• Student Summary: ALL Students
• Create Transaction: Expense, Investment, etc.
• View Transaction: ALL transactions: Income, Expense, Investment, etc.
• Financial Summary: Automatic

2. Academic

The end of this flow will be Student Progress Card.
Once data input of all three sections done for all subjects, Principal can APPROVE and "Generate Student Progress Report" -> only then the Progress Card (PDF, non editable) can be seen by teachers and parents.

Data input for Preschool is per student. Data input for Primary is per subject.

• (Primary) Data Entry for Student Progress Card Section 1: Quantitative
• (Primary) Data Entry for Student Progress Card Section 2: Qualitative (Proficiency)
• (Primary) Data Entry for Student Progress Card Section 3: Comment
• Approve Scores for Student Progress Card: (for Principal)
• Student Progress Card: Per Student
• Academic Summary: ALL Students

3. Payroll

The end of this flow will be salary summary per month (payroll is calculated automatically).

• Employee profile, position, user roles is in the same database for Payroll system and SMS, if not, AUTO sync
• Employee profile: information, salary & allowance, attendance,
• Sync from fingerprint attendance machine - attendance
• Create Payroll Slip
• Salary Summary per Month
• Salary Summary per Year
